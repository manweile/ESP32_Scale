10. Tick output rate-limiting — highest loop() throughput impact
tickCalibration() and tickLevelRead() can emit a serial write every loop() iteration during SETTLING/READING states. 
A static uint32_t lastPrintMs gate (e.g., 250 ms) collapses dozens of writes per second down to 4, 
with no perceptible difference to a human reading the monitor. 
The HX711 read cadence is the real bottleneck there, not the display rate.

11. Serial.availableForWrite() guard on large prints
The 192-byte warnBuf strings and multi-line calibration prompts can stall loop() if the 256-byte ESP32 UART TX FIFO is already partially full. 
Checking Serial.availableForWrite() >= len before the print keeps the main loop from blocking. 
Low code complexity; directly improves responsiveness during calibration flows.

12. Consolidate per-loop status output into a single structured line
Rather than emitting weight readings as prose ("Weight: 12.34 lbs\n"), 
emit a single fixed-format CSV or JSON line (e.g., W,12.34,45\n). 
One write, smaller payload, faster drain. 
Also directly reusable as the data format for the web interface's WebSocket push
— you won't have to redesign the output format mid-port.
