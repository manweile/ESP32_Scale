test criteria:
Startup auto-tare success path
Power on with the scale in its expected empty condition.
Pass: startup tare begins automatically, detects stable empty scale, applies tare, and continues to normal operation without user intervention.

Startup skip path
Power on with weight on scale (causes the startup tare wait state), then send the skip input (q) if your workflow supports it.
Pass: startup tare exits cleanly and system proceeds to the same post-startup state as before the refactor.

Startup timeout path
Power on with the scale not meeting the empty/stable condition and do not send input.
Pass: workflow times out exactly as before, prints the same timeout/skip guidance, and returns control without hanging.

Unstable scale path
Power on while the scale reading is noisy or changing.
Pass: startup tare stays in the waiting logic until stable, skipped, or timed out; it does not falsely tare during unstable readings.

HX711 not-ready path
Start with HX711 disconnected, unpowered, or otherwise not ready.
Pass: workflow reports the same diagnostics/cancel behavior as before and does not lock up.

Post-tare normal operation
After a successful startup tare, run a normal command such as level read or another existing serial command.
Pass: system behaves normally, confirming startup tare left state cleanly initialized.

Reboot repeatability
Power cycle and repeat startup tare multiple times under the same empty-scale setup.
Pass: behavior is consistent across runs, with no intermittent stuck state or missed transition.

Message parity check
Compare serial output during success, skip, and timeout runs against pre-refactor behavior.
Pass: user-visible prompts and results are unchanged unless you intentionally modified them.

1. Automatic Calibration Flow

Start with the scale empty.
Send the command to begin automatic calibration (e.g., a).
Confirm prompts: “Automatic calibration mode”, “Remove all weight from scale”, “Auto-detect is active”, etc.
Place the known calibration weight when prompted.

pass

2. Manual Calibration Flow

Start with the scale empty.
Send the command to begin manual calibration (e.g., m).
Confirm prompts: “Manual calibration mode”, “Remove all weight from scale”, etc.
Place the known calibration weight when prompted.
Use + and - to adjust the calibration factor; confirm the reading and factor update in the output.
Use s to save; confirm: “Manual calibration complete, computed calibration factor: ...”, “Calibration saved to EEPROM.”
Power cycle and send e to verify the calibration factor persisted.

 pass
3. Re-zero Flow

Start with the scale empty.
Send the command to begin re-zero (e.g., r).
Confirm prompts: “Runtime re-zero requested”, “Remove all weight from scale”, etc.
Wait for auto-confirm or send z to force.
Confirm: “Scale re-zero complete.”
Power cycle and verify the tare offset is applied (e.g., by reading a known empty value).

pass

4. Cancel Behavior

Start any calibration workflow (a, m, or r).
While in WAIT_EMPTY or WAIT_LOAD, send q.
Confirm: “Automatic calibration cancelled.” or “Manual calibration cancelled. Changes were not saved.” or “Runtime re-zero cancelled.”
Confirm state returns to idle and no calibration/tare values are changed.

pass

5. Timeout and Error Handling

Start a calibration workflow and do not interact (leave scale non-empty or do not place weight).
Wait for timeout.
Confirm: “Confirmation timed out: scale not empty; cancelled.” or “Weight placement timed out; calibration cancelled.”
Confirm state returns to idle.

pass
pass

6. Invalid Key Handling

During WAIT_EMPTY or WAIT_LOAD, send an invalid key (not q, z, +, -, or s).
Confirm: “Invalid ... calibration key: ...” message appears and workflow continues.

pass

7. Persistence and Consistency

After any calibration, power cycle and verify with e that the calibration factor is correct.
Repeat calibration and re-zero multiple times to confirm no stuck states or missed transitions.
